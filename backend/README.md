# Application Platform Backend

Simple Hello World API with Swagger documentation.

## Project Structure

```
backend/
├── cmd/
│   └── api/
│       └── v1/
│           └── main.go          # Main application
├── internal/
│   └── database/
│       └── database.go          # Database functions
├── migrate/
│   └── migrations/
│       └── main.go              # Migration script
├── docs/                        # Generated by swag (don't create manually)
└── go.mod                       # Dependencies
```

## Setup & Run

1. **Navigate to your backend folder:**
   ```powershell
   cd C:\Users\duykh\Documents\homework\BP\application-platform\backend
   ```

2. **Install dependencies:**
   ```powershell
   go mod download
   go install github.com/swaggo/swag/cmd/swag@latest
   ```

3. **Generate Swagger documentation:**
   ```powershell
   swag init -g cmd/api/v1/main.go
   ```

4. **Run the application:**
   ```powershell
   go run cmd/api/v1/main.go
   ```

## Access

- **API Endpoint:** http://localhost:8080/api/v1/hello
- **Swagger UI:** http://localhost:8080/swagger/index.html

## Test the API

Open a new terminal and run:
```powershell
curl http://localhost:8080/api/v1/hello
```

Or visit in your browser: http://localhost:8080/api/v1/hello

Expected response:
```json
{
  "message": "Hello World!"
}
```

## Optional: Run Migration

If you want to set up the database:
```powershell
go run migrate/migrations/main.go
```

## Troubleshooting

If you get "no Go files" error:
- Make sure you're in the `backend` directory
- Use the full path with `-g` flag: `swag init -g cmd/api/v1/main.go`
- Make sure all `.go` files have proper content (no empty files)
